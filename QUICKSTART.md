# ⚡ Quick Start — Creating a New Repository in Coding Pirates Denmark

Velkommen!  
Denne korte guide hjælper dig med hurtigt at oprette et repository i Coding Pirates Denmarks GitHub-organisation – og følge de fælles navngivningsregler.

Hvis du har travlt, så er dette alt du behøver.

---

# 1️⃣ Navngiv dit repository korrekt

Brug formatet:

```
<kategori>-<underemne>-<projekt>
```

**Eksempler:**
- `games-pygame-platformer`
- `hardware-microbit-ledgame`
- `web-react-clicker-game`

**Husk:**
- kun små bogstaver
- bindestreger (`-`) mellem ord
- ingen mellemrum eller specialtegn
- engelske navne

---

# 2️⃣ Vælg det rigtige kategoripræfiks

Alle repos starter med ét af disse præfikser:

| Område | Præfiks |
|-------|---------|
| Spil | `games-` |
| Hardware/elektronik | `hardware-` |
| Web | `web-` |
| Robotter | `robotics-` |
| Kunstig intelligens | `ai-` |
| Ressourcer | `resources-` |
| Skabeloner | `templates-` |
| Organisationens dokumenter | `admin-` |
| Organisationsregler (som dette repo) | `org-` |

---

# 3️⃣  Vælg passende underemner
Underemnet beskriver teknologien eller platformen.

Eksempler:

- `pygame`
- `kaplay`
- `godot`
- `microbit`
- `raspberrypi`
- `arduino`
- `htmlcss`
- `python`
- `js`
- `react`

# 4️⃣ Vælg projektnavn

Projektdelen beskriver projektets titel, fx:

- `streetfighter`
- `platformer`
- `space-shooter`
- `sensor-kit`
- `basic-site`
- `imageclassifier`

Hold det kort, klart og uden specialtegn.

# 5️⃣ Tilføj GitHub Topics

Mindst disse:

- `coding-pirates`
- én kategori-topic (fx `games`, `hardware`, `web`, …)

**Valgfrie ekstra topics:**
- teknologi: `python`, `pygame`, `godot`, `react`, `microbit`, …  
- niveau: `beginner-friendly`, `intermediate`  
- indholdstype: `lesson`, `project`, `exercise`  

**Eksempel:**

`coding-pirates`, `games`, `pygame`, `python`, `beginner-friendly`

---

# 6️⃣ Brug en god README

Tilføj mindst:

- projektets titel  
- kort beskrivelse  
- installation eller startvejledning  
- hvilke teknologier der bruges  
- hvem repoet er til  

(Dette gør det meget nemmere for andre frivillige at forstå projektet.)

---

# 7️⃣ Tjekliste inden du opretter repoet

✔ Har du brugt korrekt navngivning?  
✔ Har du valgt det rigtige kategoripræfiks?  
✔ Er projektnavnet kort og tydeligt?  
✔ Har du tilføjet relevante Topics?  
✔ Er README’en klar til brug?  

---

# 💬 Brug for hjælp?

Hvis du er i tvivl:

- spørg en anden frivillig  
- opret et Issue i `org-repo-naming`  
- eller fang os på Slack afhængigt af din lokale afdeling  

Tak fordi du hjælper med at holde vores organisation overskuelig og inspirerende! 🏴‍☠️
